hexo博客的搭建

Posted by cc-tian on 2015-11-18

意义

为什么技术员要写博客?

首先，为自己，强化知识，提升学习能力，提升语言组织能力，提升逻辑思维能力；其次就是为他人，我们平时开发项目的时候会遇到问题，就会去百度，然后看别人博客就轻而易举的解决了问题，而且还会搜索很多技术博客去学习其中的思想和技术，促进自己的进步，所以我们不单单只是去获取，可以为他人提供自己的经验和学习经历，或许别人碰到的问题看了你的博客后也能够轻易的解决。
虽然hexo不是世界上最流行的博客平台，但是由于hexo基于node的博客框架，还可以实现基于git的博客，拥有免费的服务器空间。相比workpress来说更轻更快，更适合node开发程序员。把博客从wordpress迁移到github，还要学Ruby和Jekyll。而且迁移过程过于复杂，所以hexo正在慢慢流行。

项目构建

用 hexo 构建一个初始项目

  1. 先用全局安装: npm install hexo–g
  2. 初始化一个工程: hexo init [工程名]
  3. 安装hexo-server包: npm install
  4. 生成一篇新的博客:hexo new [文章名]
  5. 修改主题,将主题下载到themes目录下:git clone [仓库https协议地址]
  6. 修改_config.yml配置文件下的theme属性值， 默认为theme: landscape
    将其值修改为刚刚下载的主题名即可，然后就可以开启服务器预览了。
  7. 生成静态文件并监视文件变动: hexo g -w
  8. 启动hexo服务器进行本地预览: hexo s
  9. 打开浏览器: http://localhost:4000/

github配置

1. 注册git账号

2. 创建仓库

创建仓库的时候仓库名最好是 [用户名].github.io 的命名格式

3. 本地git环境配置

绑定git账号： git config –global user.name “用户名”
绑定git邮箱： git config –global user.email “邮箱”

4. 克隆仓库到本地

git clone [仓库的URL]

5. 创建ssh密钥

ssh-keygen -t rsa –C [注册邮箱]

SSH 公钥默认储存在账户的主目录下的 ~/.ssh 目录

$ cd ~/.ssh
$ ls
id_dsa   id_dsa.pub   known_hosts

打开生成的xx.pub文件，复制内容，找到git用户的settings选项，选择SSH key，Add ssh key,保存OK!

静态化工程

hexo generate

注意：命令必须在工程根目录下执行，否则将找不到node_modules,无法加载相应模块
通过静态化命令，将工程转化成网页静态文件，即js,html文件

下载hexo-deployer-git模块

npm install hexo-deployer-git –save

部署到git仓库

hexo clean && hexo g && hexo d

部署完后我们可以在仓库settings里看到GitHub Pages一栏下看到：
Your site is published at http://xxx.github.io
这么一句话，后面的url地址就是git默认给你生成的一个访问网址，通过这个url就可以在浏览器访问你的站点了。
但是通过这个链接访问的项目加载不到样式，所以我们还需要绑定域名。

最后在gh-pages分支下的项目根目录添加一个文件CNAME,里面填写你的域名。
